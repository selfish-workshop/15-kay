## 기능 명세: 빠른 코멘트 (Quick Comment)

### 문제 정의

실제 운동 기록 15세션(2026-02 ~ 2026-04)을 분석한 결과, 세트 간 코멘트 입력에서 다음 문제가 발견되었다:

1. **반복 패턴**: "힘듦", "어깨 쓰임", "반동 씀", "무릎 찌릿함" 등 거의 매번 비슷한 표현을 타이핑
2. **오타 빈발**: 땀에 젖은 손으로 입력 → "세쎄트", "힘들었거", "느낌이 남" 등 오타 다수
3. **시간 소요**: 30~90초 휴식 중 타이핑에 시간을 소모하면 세트 간 리듬이 깨짐
4. **데이터 비정형**: 자유 텍스트는 AI가 파싱하기 어렵고, 동일 의미를 다른 표현으로 기록

### 해결 방식

사전 정의된 **태그 칩(chip) 버튼**을 탭하여 코멘트를 완성하는 원터치 입력 시스템. 전체 과정 **5초 이내** 완료 목표.

### UX 흐름

기존 "② 운동 수행 + 코멘트 작성" 단계의 코멘트 입력부를 개선한다.

```
세트 완료 체크 (무게/횟수 입력)
    │
    ▼
┌──────────────────────────────────────────────┐
│  코멘트 입력 영역                               │
│                                              │
│  ┌─ 빠른 태그 (카테고리 탭 좌우 스와이프) ──────┐  │
│  │  체감 │ 자극 │ 통증 │ 폼 │ 무게 │ 완수 │ ♥  │  │
│  │  ────────────────────────────────────    │  │
│  │  ○ 가벼움  ● 적당함  ○ 힘듦  ○ 개힘듦     │  │
│  └──────────────────────────────────────────┘  │
│                                              │
│  선택됨: 적당함 · 어깨 너무 쓰임 · 반동 씀  [×]    │
│                                              │
│  ┌──────────────────────────────────────────┐  │
│  │ 추가 메모... (자유 텍스트, 선택사항)         │  │
│  └──────────────────────────────────────────┘  │
│                                              │
│           [ 건너뛰기 ]    [ 저장 ]              │
└──────────────────────────────────────────────┘
```

**핵심 원칙**:
- 코멘트 영역 오픈 시 빠른 태그가 먼저 보임 (자유 텍스트는 접힌 상태)
- 태그 탭 1회로 토글 (선택/해제)
- 여러 카테고리에서 복수 선택 가능 (조합)
- 자유 텍스트는 태그와 병행 가능
- "건너뛰기"로 코멘트 없이 넘어갈 수 있음

### 태그 카테고리 정의

실제 운동 기록에서 추출한 7개 카테고리:

#### A. 난이도/체감 (단일 선택)

| 태그 | 의미 |
|------|------|
| 너무 가벼움 | 무게 올려야 할 수준 |
| 할만함 | 적정 난이도 |
| 적당함 | 약간 힘들지만 폼 유지 가능 |
| 힘듦 | 상당히 힘들었음 |
| 개힘듦 | 거의 한계 |
| 겨우 함 | 짜내서 겨우 완수 |

> 실제 기록: "30은 좀 힘듦", "할만해서 올림", "개개개 힘듦", "마지막 회차때 다리가 힘빠지고"

#### B. 타겟 근육 자극 (2단계 선택, 복수 가능)

**1단계 — 부위 선택:**
가슴 | 등 | 어깨 | 삼두 | 이두 | 코어 | 앞허벅지 | 뒷허벅지 | 엉덩이 | 팔(전체)

**2단계 — 느낌 선택:**
| 태그 | 의미 |
|------|------|
| 잘 느껴짐 ✓ | 목표 근육에 자극 OK |
| 안 느껴짐 ✗ | 자극이 약하거나 없음 |
| 너무 쓰임 ⚠ | 비목표 근육이 과도 개입 |
| 힘 빠짐 ↓ | 근피로로 힘이 빠짐 |

> 실제 기록: "어깨가 더 쓰임", "가슴근육 인지 어렵다", "삼두 힘 빠짐", "겨드랑이 쪼이는 느낌"

**조합 예시**: `어깨` + `너무 쓰임` → "어깨 너무 쓰임"

#### C. 통증/불편 (2단계 선택, 복수 가능)

**1단계 — 부위:**
무릎 | 손목 | 팔꿈치 | 허리 | 목 | 어깨 | 머리 | 속/배

**2단계 — 증상:**
| 태그 | 의미 |
|------|------|
| 찌릿함 | 날카로운 신경성 자극 |
| 걸리는 느낌 | 관절이 걸리거나 뚝뚝 |
| 어지러움 | 핑 돌거나 현기증 |
| 울렁임 | 속이 메스꺼움 |
| 압박감 | 눌리는 느낌 |
| 뻐근함 | 묵직하게 뻐근 |
| 꺾이는 느낌 | 관절이 꺾이는 불편감 |

> 실제 기록: "왼무릎 동그란 뼈 윗쪽 아주 약한 찌릿함", "머리가 핑돌아서 멈춤", "속이 약간 울렁임", "팔꿈치가 걸리는 느낌"

**⚠ 중요**: 통증 태그 선택 시 AI 피드백 엔진에 **우선 플래그** 전달. 동일 부위 통증 3회 반복 시 운동 중단 권고.

#### D. 자세/폼 (복수 선택)

| 태그 | 의미 |
|------|------|
| 반동 씀 | 목표 근육 외 반동으로 수행 |
| 자세 무너짐 | 폼이 흐트러짐 |
| 몸 비틀어짐 | 좌우 비대칭 움직임 |
| 좌우 불균형 | 한쪽이 명확히 약함 |
| 궤적 불안정 | 동작 궤적이 일정하지 않음 |
| 가동범위 부족 | 끝까지 못 벌리거나 못 접음 |

> 실제 기록: "자꾸 반동으로 하려고 함", "왼쪽이 안올라가서 몸 비틀게 됨", "궤적이 동일하지 않음"

#### E. 무게 조정 사유 (단일 선택)

| 태그 | 의미 |
|------|------|
| 가벼워서 올림 ↑ | 계획 무게가 너무 가벼움 |
| 무거워서 내림 ↓ | 계획 무게가 너무 무거움 |
| 폼 유지 위해 내림 | 자세 무너짐 방지 |
| 통증으로 내림 | 통증/불편으로 감량 |

> 실제 기록: "5는 너무 가벼워서 10으로", "자세때문에 15로 낮춰서 함", "너무 무거워서 앞 어깨근육이 더 쓰인듯"

#### F. 세트 완수도 (단일 선택)

| 태그 | 의미 |
|------|------|
| 계획대로 ✓ | 계획한 횟수/무게 그대로 |
| 더 함 ++ | 계획보다 더 수행 |
| 겨우 함 ~ | 힘들었지만 채움 |
| 못 채움 ✗ | 목표 횟수 미달 |

> 실제 기록: "계획대로", "20개 했어", "마지막은 겨우 짜내서", "까먹고 못함"

#### G. 심박수 (숫자 입력)

별도 숫자 스텝퍼 UI (+/- 버튼, 5bpm 단위). 범위: 60~220 bpm.

> 실제 기록: "심박수 120-122사이", "심박 133까지 감", "심박 136 찍음"

### 태그 + 자유 텍스트 결합

선택된 태그는 **자연어 문장으로 렌더링**되어 AI 피드백 엔진과 히스토리 오버레이에 전달된다.

**렌더링 규칙** (순서대로 조합):
1. 난이도 → 2. 자극 → 3. 통증 → 4. 폼 → 5. 무게 사유 → 6. 완수도 → 7. 자유 텍스트

**예시**:
사용자가 `힘듦` + `어깨·너무 쓰임` + `반동 씀` + 자유 텍스트 "마지막 렙 짜냄" 선택 시:
→ `"힘듦. 어깨 너무 쓰임. 반동 씀. 추가: 마지막 렙 짜냄"`

### AI 엔진 연동

태그 데이터는 구조화되어 있으므로 AI가 자연어 파싱 없이 정확한 시그널로 활용 가능.

| 트리거 | 동작 |
|--------|------|
| 통증 태그 선택 | **즉시 피드백** — 안전 가이드 표시 ("무릎 찌릿함 → ROM을 줄이고 무게 10% 감량 권장") |
| 코멘트 저장 | **세트 간 피드백** — 히스토리 오버레이에 AI 코멘트 추가 |
| 동일 이슈 3회 반복 | **패턴 알림** — "최근 3회 벤치프레스에서 '어깨 쓰임' 반복. 그립 너비 조정을 시도해보세요" |
| 세션 종료 | **운동 완료 요약** — 전체 태그 종합 분석, 다음 세션 추천 |

### 스마트 추천 (v2)

| 추천 유형 | 설명 |
|----------|------|
| 운동별 태그 필터 | 벤치프레스 → 자극 카테고리에 가슴/삼두/어깨 우선 표시 |
| 히스토리 기반 | 자주 선택하는 태그를 "자주 쓰는 태그" 섹션으로 상단 노출 |
| 세트 번호 기반 | 1세트: 난이도 우선 / 마지막 세트: 완수도+무게사유 우선 |
| 커스텀 태그 | 자유 텍스트 3회 이상 유사 입력 시 "태그로 추가?" 제안 |
| 프리셋 저장 | 자주 쓰는 태그 조합을 "프리셋"으로 저장 (최대 3개) |

### MVP 범위

| 항목 | 버전 |
|------|------|
| 6개 카테고리 기본 태그 (난이도, 자극, 통증, 폼, 무게사유, 완수도) | **MVP** |
| 심박수 숫자 입력 | **MVP** |
| 태그 + 자유 텍스트 병행 | **MVP** |
| 자연어 렌더링 및 AI 전달 | **MVP** |
| 통증 태그 즉시 피드백 | **MVP** |
| 운동별 스마트 태그 필터 | v2 |
| 히스토리 기반 추천 | v2 |
| 커스텀 태그 / 프리셋 | v2 |

### 설계 시 고려사항

- **한 손 조작**: 모든 칩은 엄지 도달 범위(화면 하단 2/3) 내 배치. 카테고리 전환은 좌우 스와이프.
- **터치 영역**: 칩 최소 44×44pt, 칩 간 간격 8pt 이상 (땀에 젖은 손, 장갑 착용 고려)
- **오프라인**: 태그 데이터 로컬 저장 → 온라인 복귀 시 동기화. 통증 알림만 로컬 룰 기반 오프라인 동작.
- **프리미엄 구분**: 태그 입력 자체는 **무료** (입력 편의는 기본 가치). AI 피드백 연동·스마트 추천은 **프리미엄**.
